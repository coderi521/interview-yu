MCP 全称是 Model Context Protocol，我把它理解为“**AI Agent 和外部世界交互的标准插座**”。

它主要解决一个问题：过去我们让模型去调用数据库、业务系统或第三方 API，都要为每个接口单独写一套适配逻辑、单独教模型怎么用。久而久之，接口风格不统一、维护成本也很高。

MCP 做的事情，就是用一个开源的标准协议，把“工具长什么样、有哪些参数、怎么被调用、返回什么”全部结构化描述出来，然后用类似 JSON-RPC 的方式统一发起调用和拿结果。对模型来说，就是统一通过 `tools.call` 之类的方法去“调函数”；对后端来说，只要把能力封装成 MCP 工具，就能被各种 Agent/模型复用。

所以从工程视角看，**MCP 把‘模型和工具之间的桥’标准化了**：一方面解耦了模型和具体系统，另一方面也方便做权限、日志和监控。现在很多 Agent 平台和 IDE 都在往 MCP 靠拢，用它做为连接模型和企业系统的基础设施。
